Extra's

Tijdens het ontwikkelen van de Mule-applicatie en het portfolio, ben ik nog in aanraking gekomen met een aantal nieuwe technologieën die ik toch ook nog het vermelden waard vond. Deze tellen niet mee voor de hoofdopdracht en zijn ook niet groot genoeg om als volledige technologie mee te kunnen tellen, maar ik wilde deze toch ook nog even licht aanraken in mijn portfolio.

PostgreSQL en Portainer

Voor mijn databases heb ik gebruik gemaakt van Docker-containers om deze in te doen draaien. Dit zijn PostgreSQL-databases, wat ook een nieuw technologie is waarmee ik nog niet bekend was. Ik heb Portainer in een Docker-container opgezet (en ook een Docker volume) en vervolgens via de Portainer GUI heb ik twee databases aangemaakt. Hier kon ik zeer eenvoudig gegevens zoals de naam van de database, de gebruiker en het wachtwoord instellen.

Vervolgens maakte ik via DBeaver connectie met de databases en gaf ik via een script gegevens in, in de database. De syntax lijkt enorm hard op SQL, of toch wat ik nodig had. Ik heb geen gebruik gemaakt van joins of iets anders advanced.

Reflectie

Wanneer ik besloten had enkele databases te voorzien, had ik besloten om ook hier maar eens naar enkele nieuwe technologieën te kijken. Portainer had ik al eens van gehoord, maar tot nu toe nog nooit gebruikt. Dit vond ik dus het ideale moment.

Ook PostgreSQL heb ik al vaker gehoord maar nog nooit gebruikt. Voor zo een klein project vond ik dat ideaal om eens iets nieuws uit te proberen.

Ik moet toegeven dat ik dit niet extreem moeilijk vond. Tegenwoordig zijn er heel wat Docker images beschikbaar en is installeren gebeurd met slechts enkele commando’s. Ook het opzetten via Portainer was heel eenvoudig. Ik heb dus geen problemen ondervonden.

Ik heb besloten deze technologieën ook nog even te vermelden in het portfolio omdat deze toch ook deel hebben uitgemaakt van het gehele plaatje.

GitHub Pages

Bij het aanmaken van GitHub Pages heb ik geen problemen ondervonden. In het begin was het mij niet duidelijk dat de website achterliggend opnieuw gebuild moest worden, waardoor mijn wijzigingen niet onmiddellijk zichtbaar zijn op de website. Daarbuiten ben ik niets abnormaals of moeilijks tegengekomen, GitHub Pages werkt enorm gemakkelijk.

Ik vond de manier van werken enorm gemakkelijk. Iemand met basiskennis van Git, HTML en CSS kan op deze manier heel gemakkelijk en gratis een website maken en hosten. Uitgebreide kennis is absoluut niet nodig.

In de opleiding hebben we regelmatig gewerkt met Git en GitHub, en om dat te combineren met het effectief hosten van een website zonder extra werk, vond ik enorm verrassend en leuk om te ontdekken. Ik word enorm blij van simpele dingen die werken zoals ze moeten werken, zonder addertjes onder het gras. Ik had in het begin wat schrik omdat ik nog niet echt ervaring had met het hosten van gratis websites. Ik had verwacht dat dit een heel gedoe ging worden met veel opzoekwerk, maar dat bleek dus allesbehalve.

Ik heb deze opdracht gekozen om in het portfolio te steken, omdat ik enorm verrast ben dat ik heb bijgeleerd dat het hosten van een gratis website zo gemakkelijk kan zijn.
